Acceptance criteria
- POST/PUT for industries & stages return the created/updated DTO and FE updates local state without full reload.
- DELETE endpoint returns 200 + ApiResponse<string> describing action (hard-delete or soft-delete). FE shows toast with message.
- When BE indicates soft-delete, FE marks item as `isActive = false` (keeps record in UI). When BE indicates hard-delete, FE removes the item from the tree.
- Confirm dialogs replaced by UI modal showing `startupCount` and `investorCount` when present.
- Restoring a parent does NOT auto-restore child industries (BE rule). FE should not attempt to auto-restore children.
- DTO fields use camelCase; FE consumes `industryId`, `parentIndustryId`, `subIndustries`, `stageId`, `createdAt`, `updatedAt`, `startupCount`, `investorCount`.

2. Create a new industry (no parent) via "Thêm ngành" — expect the new item appears in the list without full page reload.
3. Edit an industry and save — expect updated fields reflected inline immediately.
4. Create a sub-industry (select parent) and verify it appears under parent.
5. Attempt to hide an industry that has startup/investor usages — confirm modal shows counts; confirm action either soft-delete (item remains in list with "Đã ẩn") or hard-delete (item removed). Verify toast message from BE is shown.
6. Restore an industry and verify `isActive` toggles to true; children should not be automatically restored.
7. For stages: create/edit/hide/restore similarly; test `orderIndex` behavior.
8. Verify `createdAt`/`updatedAt` are available in DTO (for use in details/history if needed).
